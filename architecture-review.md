# Architecture & Design Review

## 1. Frontend Architecture
- **Design Origin:** The frontend uses high-fidelity HTML/Tailwind mockup screens ("Alloy Desktop" style) generated by the Stitch MCP.
- **Delivery Mechanism:** The screens (`data_source.html`, `forecasting.html`) are served statically over HTTP using a `FastAPI` static mount (`/ui`).
- **Review:** This is extremely lightweight. By avoiding a full Node.js/React build pipeline, we can keep the packaging simple and focus purely on getting the Python layer to run perfectly on the Mac.

## 2. Backend Architecture
- **Data Engines:** `causal_engine.py` and `forecasting_engine.py` remain as Python modules, imported into `api/routes.py`.
- **API Layer:** `FastAPI` (running via `uvicorn`) exposes the data pipelines over local HTTP.
- **Review:** This decoupling ensures the Python heavy-lifting (CausalNex, Scikit-learn, PyTorch) runs efficiently natively on the host Mac, while the frontend is just a thin presentation layer. This is industry standard for bridging web UI with local compute (similar to Electron, but much lighter).

## 3. Native App Wrapper (The "Think Twice" Moment)
- **Framework:** `pywebview` utilizing macOS's native `WKWebView`.
- **Vibrance Effect:** By starting the application with `transparent=True` and `frameless=True`, macOS automatically applies its native hardware-accelerated Vibrancy blur behind our translucent web elements.
- **"Think Twice" Concern - Bundling & Paths:** When PyInstaller creates the `.app` bundle, the application extracts its contents into a temporary directory (accessed via `sys._MEIPASS`). If we rely on standard `__file__` paths to find `src/ui/frontend`, the API server will crash inside the `.app` bundle.
- **Resolution:** We have modified `src/api/routes.py` to check for `sys._MEIPASS` before falling back to standard `__file__` resolution. We will also pass `--add-data "src/ui/frontend:src/ui/frontend"` to PyInstaller to ensure the HTML files are bundled in the `.app`.

## 4. Security
- **Localhost Only:** The FastAPI server is strictly bound to `127.0.0.1`.
- **Review:** Since it's a desktop app, binding to localhost is correct, preventing other machines on the local network from accessing the user's data sources.

## Conclusion
The architecture is solid, performant, and correctly tailored for macOS. We are ready to proceed to packaging via PyInstaller.
